# 接口文档

## GET /

返回应用名称、运行状态和当前 LLM provider。

## POST /api/documents/upload

上传文档。支持 `.txt`、`.md`、`.pdf`、`.docx`。

表单字段：

- `file`: 上传文件

## GET /api/documents

返回已上传文档列表。

## POST /api/knowledge/build

构建知识库。

请求体：

```json
{"doc_ids": []}
```

`doc_ids` 为空时构建全部文档。

重复构建同一文档时，会先删除该文档旧 chunks 和旧向量，再写入新 chunks。

## GET /api/knowledge/stats

返回文档数量和 chunk 数量。

## DELETE /api/knowledge/clear

清空知识库，删除 SQLite chunks 表和向量库中的 chunk 向量，但不删除原始上传文档。

返回示例：

```json
{
  "code": 0,
  "message": "knowledge base cleared successfully",
  "data": {
    "deleted_chunks": 120
  }
}
```

## POST /api/knowledge/rebuild

重建指定文档。

请求体：

```json
{"doc_ids": ["xxx"]}
```

系统会重新解析文档、切分文本、向量化并写入向量库。

## DELETE /api/documents/{doc_id}

删除指定文档及其 chunks。

功能包括：

- 删除 SQLite documents 记录；
- 删除 SQLite chunks 记录；
- 删除向量库中该文档对应向量；
- 删除 `data/raw_docs` 中的原始文件。

## POST /api/chat

普通 RAG 问答接口。

请求体：

```json
{"query": "什么是 RAG？", "top_k": 5}
```

返回中的 `sources` 包含 `doc_id`、`filename`、`chunk_id`、`chunk_index`、`score` 和截断后的 `text`。

## GET /v1/models

OpenAI-compatible 模型列表接口，供 OpenWebUI 读取模型。

## POST /v1/chat/completions

OpenAI-compatible 聊天接口，内部调用 RAG chain。

请求体示例：

```json
{
  "model": "rag-mcp-knowledge-platform",
  "messages": [
    {
      "role": "user",
      "content": "什么是 RAG？"
    }
  ],
  "stream": false
}
```

说明：

- 后端会读取最后一条 `role=user` 的消息作为用户问题；
- `messages` 为空或没有 user 消息时返回 400；
- `stream=true` 时当前阶段返回基础 SSE，保证 OpenWebUI 不报错；
- 内部会先执行 RAG 检索，再根据 `LLM_PROVIDER` 调用 mock、API 或 Ollama。
